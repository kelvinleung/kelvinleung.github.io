Axios 源码阅读笔记之——徒手撕 Ajax（一）

DEC 30, 2021

花了两三天的时间，把 Axios 的源码过了一遍。不得不说，Axios 的源码确实很适合小白，逻辑不复杂，也能看到很多巧妙的地方。看一遍，然后自己把核心的逻辑实现一遍，确实能学到很多东西，在此记录一下。

说到 Axios，不得不说 Ajax，实际上 Axios 就是一个封装 Ajax 请求的库。而 Ajax 则是 Asynchronous JavaScript and XML 的缩写。这一术语最早出现在 2005 年 Jesse James Garrett 的一篇文章中。

要知道，在盘古初开的时候，使用 JS 去动态请求数据，一般只能用<form>的action来向指定 URL 发起请求，并且这会导致整个页面刷新，体验非常的差。后来微软为了优化 Web 版的 Outlook 体验，在 IE 中增加了XMLHttpRequest，令到 JS 能够异步请求数据。其它浏览器厂家纷纷跟进，分别实现了自己版本的XMLHttpRequest功能，随后更是被纳入了 W3C 规范中。当然，将这项技术发扬光大的是我们熟知的 Gmail 与 Google Maps，并由此打开了 Web 2.0 时代的大门。

以上这段历史感兴趣的可以看下这篇文章。

值得注意的是，Ajax 这一术语，并不是单单指某一项技术，而是一组技术的组合运用，MDN 中对它的定义描述为：

Asynchronous JavaScript and XML, while not a technology in itself, is a term coined in 2005 by Jesse James Garrett, that describes a "new" approach to using a number of existing technologies together, including HTML or XHTML, CSS, JavaScript, DOM, XML, XSLT, and most importantly the XMLHttpRequest object. When these technologies are combined in the Ajax model, web applications are able to make quick, incremental updates to the user interface without reloading the entire browser page. This makes the application faster and more responsive to user actions.

而在狭义的语境中，Ajax 一般指的就是“发起异步请求”。使用 JS 发起一个 Ajax 请求很简单，只是有点繁琐，代码如下：

code
1const xhr = new XMLHttpRequest();2
3// 初始化请求4xhr.open("GET", "https://www.xxx.com/api");5// 设置请求头6xhr.setRequestHeader("X-Requested-With": "XMLHttpRequest");7// 监听请求状态8xhr.onreadystatechange = function () {9  if (xhr.readyState !== 4) {10    return;11  }12  if (xhr.status >= 200 && xhr.status < 300) {13    console.log(xhr.responseText);14  }15};16// 发送请求17xhr.send();

每次请求都需要如此繁琐的配置，实在没有必要。接下来我们从零开始，封装一个简单易用的 Ajax 请求库，实现简化版的 Axios。

面向调用设计

之前做产品的时候，当我们需要去设计一个功能时，总是会提到“以用户为中心”，“面向用户”这些术语。因为产品最终是要交付给用户使用，才会产生价值。因此要面向用户的使用场景、需求去设计。而编写一个库，同样需要面向“用户”设计，这个用户，指的是这个库的调用者。因此当我们设计 API 时，要考虑的是调用方是如何调用的。

那么作为调用者，我们希望如果调用这个简化版的 Axios 呢。当然是越简单越好，最好是能不设置的就不用设置，直接就可以发起请求。同时，能提供多种调用方式，以满足不同调用者的使用习惯。Axios 的调用方法就非常多样，例如：

code
1// 默认情况下直接发起 GET 请求2axios("https://www.xxx.com/api").then((res) => console.log(res.data));3
4// 使用 get 方法发起请求5axios.get("https://www.xxx.com/api").then((res) => console.log(res.data));6
7// 指定参数8axios({9  method: "get",10  url: "https://www.xxx.com/api",11}).then((res) => console.log(res.data));12
13// 附加指定参数14axios("https://www.xxx.com/api", { method: "get" }).then((res) =>15  console.log(res.data)16);17
18// 还可以通过实例去调用19const instance = axios.create({20  baseURL: "https://www.xxx.com/api",21  timeout: 1000,22  headers: { "X-Custom-Header": "foobar" },23});24// 实例方法25instance.get("login");

小结一下就是：

  • 作为函数直接调用
  • 作为对象方法调用
  • 可以创建实例，通过实例方法调用
  • 使用 Promise 链式调用
  • 方法的参数有多种形式
  • 提供缺省的默认参数

函数与对象

Axios 既可以作为函数直接调用，也可以作为一个对象，通过它的方法来调用，甚至还可以调用该对象的create()方法，创建出实例，然后通过实例的方法来调用。那么它是怎样做到的呢？

我们知道，在 JS 中，“万物皆对象”，所以即使是函数，它实际上也是一个Function对象。因此axios的本体，是一个函数（对象），这个对象（函数）还拥有像get()、create()等方法，原理如下：

code
1// axios 作为一个函数2const axios = function request(config) {3  console.log("as a function");4};5// axios 作为一个对象6axios.get = function get(config) {7  console.log("as a method of an object");8};9
10axios(); // "as a function"11axios.get(); // "as a method of an object"

由于get()方法实际上是request({ method: "get" })，因此可以用面向对象的方式来实现。简化后的代码如下：

code
1function Axios(instanceConfig) {2  // 默认的配置3  this.defaults = instanceConfig;4  // 拦截器，后面有讲，先简化省略5  this.interceptors = {};6}7
8Axios.prototype.request = function request(configOrUrl, config) {9  /**10   * 对 config 进行处理，后面有讲11   */12  // 将请求派发给对应的适配器，后面会有讲到13  return dispatchRequest(config);14};15
16["delete", "get"].forEach(function forEachMethodWithoutData(method) {17  Axios.prototype[method] = function (url, config) {18    return this.request(19      mergeConfig(config || {}, { method, url, data: (config || {}).data })20    );21  };22});23
24["post", "put"].forEach(function forEachMethodWithData(method) {25  Axios.prototype[method] = function (url, data, config) {26    return this.request(mergeConfig(config || {}, { method, url, data }));27  };28});

请求函数request()挂载在Axios的原型上，函数内部调用了dispatchRequest()函数，作用是将请求派发给对应的适配器，后面会有讲到。Axios的原型上还挂载有delete()、get()、post()、put()等常用的 HTTP 请求方法，它们被分成两组，区别是请求时是否带有data。

通过Axios构造函数，我们可以生成axios对象（函数）：

code
1function createInstance(defaultConfig) {2  const context = new Axios(defaultConfig);3  // instance 即 request4  const instance = Axios.prototype.request.bind(context);5  // 将 Axios 原型上的方法加到 instance 上6  utils.extend(instance, Axios.prototype, context);7  // 将 Axios 实例上的属性加到 instance 上8  utils.extend(instance, context);9  // instance 即为函数，又为对象10  return instance;11}12
13// 默认的参数 defaults14const axios = createInstance(defaults);15
16axios.create = function create(instanceConfig) {17  // mergeConfig 方法用于合并配置，后面有讲18  return createInstance(mergeConfig(axios.defaults, instanceConfig));19};

工具方法extend(a, b)的作用是将b对象自有的属性与方法，拷贝给a对象，简化版的实现如下：

code
1function extend(a, b, thisArg) {2  for (const key in b) {3    if (b.hasOwnProperty(key)) {4      if (thisArg && typeof b[key] === "function") {5        a[key] = b[key].bind(thisArg);6      } else {7        a[key] = b[key];8      }9    }10  }11  return a;12}

实际上axios的本体（instance）就是Axios.prototype.request()函数（对象）。它通过extend()方法获得了Axios原型上的所有方法及 Axios 实例context上的所有属性（即defaults与interceptors等属性）。它自身还增加了create()方法，该方法通过调用createInstance()方法，利用传入的配置参数，生成了新的axios对象。

默认参数

人都是懒的，为了能达到“能不配置就不配置”的目的，我们需要提供默认的参数配置项。初始化axios对象时传入的defaults对象中就包含了所有参数的默认值，如超时时间、请求头等。

code
1const defaults = {2  /*3   * 省略4   */5  timeout: 0,6  headers: {7    common: {8      Accept: "application/json, text/plain, */*",9    },10  },11};12// 初始化时传入默认参数13const axios = createInstance(defaults);

我们在发起请求时，如果不指定方法类型，默认是“GET”方法，同时，请求的参数，既可以是url，也可以是config对象，这些逻辑的处理都是在Axios.prototype.request()方法中实现的：

code
1Axios.prototype.request = function request(configOrUrl, config) {2  // 第一个参数为 string 时（url）3  if (typeof configOrUrl === "string") {4    config = config || {};5    config.url = configOrUrl;6  } else {7    config = configOrUrl || {};8  }9  // 必须要有 url，否则报错10  if (!config.url) {11    throw new Error("Url not valid");12  }13  // 设置 method 的默认值14  if (config.method) {15    config.method = config.method.toLowerCase();16  } else if (this.defaults.method) {17    config.method = this.defaults.method.toLowerCase();18  } else {19    config.method = "get";20  }21  // 合并配置22  config = mergeConfig(this.defaults, config);23  // 将请求派发给对应的适配器，后面会有讲到24  return dispatchRequest(config);25};

其中mergeConfig(a, b)方法用于合并配置，b中与a相同的配置项，会直接覆盖（b优先于a），方法的简化版实现如下：

code
1function mergeConfig(config1, config2) {2  return { ...config1, ...config2 };3}

mergeConfig()方法在很多地方有使用到，因此这些配置间会相互覆盖，它们间的优先级如下：

code
1/* 最低：不设置时，默认 timeout 为 02 * defaults = { timeout: 0 }3 */4// 低5axios.defaults.timeout = 1000;6// 中7const instance = axios.create({ timeout: 2000 });8// 高9instance.defaults.timeout = 3000;10// 最高11instance.get("https://www.xxx.com/api", { timeout: 5000 });

适配器

Axios 既可以在浏览器环境中使用，也可以在 Node 中使用，原因是使用了适配器，当我们通过dispatchRequest()方法派发请求时，会先根据不同的环境，加载不同的请求方法。其中浏览器端使用的是XMLHttpRequest，而在 Node 中则使用的是http模块。

在这里，我们先不考虑 Node 环境。但考虑到练手的需要，适配器的部分还是实现了，但直接忽略掉环境判断的部分，默认使用XMLHttpRequest适配器，简化后的代码如下：

code
1function dispatchRequest(config) {2  const adapter = config.adapter || defaults.adapter;3  return adapter(config).then(4    (response) => {5      return response;6    },7    (reason) => {8      return Promise.reject(reason);9    }10  );11}

可以看到，请求从request()到dispatchRequest()，最终是由adapter()来发起。之所以这么绕，原因一个是为了使用拦截器，这个后面再讲；另一个则是适配不同的环境。

这里再补充一点，对于环境的判断，Axios 中是利用了 Node 中的process，代码可以在源码中的lib/defaults.js文件中找到：

code
1function getDefaultAdapter() {2  var adapter;3  if (typeof XMLHttpRequest !== "undefined") {4    // For browsers use XHR adapter5    adapter = require("./adapters/xhr");6  } else if (7    typeof process !== "undefined" &&8    Object.prototype.toString.call(process) === "[object process]"9  ) {10    // For node use HTTP adapter11    adapter = require("./adapters/http");12  }13  return adapter;14}

Promisify

上面提到，最终发起请求的，是适配器，对于浏览器环境，就是xhrAdapter()，其实就是使用XMLHttpRequest来发起请求（绕了这么久，终于绕回来了）。

原生的XMLHttpRequest使用回调函数来监听各种事件，容易造成回调地狱，也不利于链式调用。我们需要将其转化为更现代、更“科学”的 Promise，这个过程，叫做 Promise 化（Promisify）。Promisify 是常见的面试题，原理很简单，当成功时resolve()，当失败/出错时reject()。我们把上面原生的 xhr 请求 Promise 化，代码如下：

code
1function xhrAdapter(config) {2  return new Promise((resolve, reject) => {3    const request = new XMLHttpRequest();4    request.open(config.method.toUpperCase(), config.url);5    request.onreadystatechange = function handleLoad() {6      if (request.readyState !== 4) {7        return;8      }9      if (request.status >= 200 && request.status < 300) {10        // 当返回码为 2xx 时即成功11        resolve(request.response);12      } else {13        // 否则为失败14        reject(request.status);15      }16    };17    request.send();18  });19}
