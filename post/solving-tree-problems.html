算法小抄——“二叉树”的套路

NOV 12, 2021

“树”是最常见的数据结构之一，与“二叉树”相关的题也是 Leetcode 中最常见的。而且大部分算法技巧，都与树的遍历有关，很多算法题，本质上就是树的遍历。二叉树的题目，都可以用下面的框架去解决：

>_<js
function traverse(root) {  // [前序遍历钩子位置]  traverse(root.left);  // [中序遍历钩子位置]  traverse(root.right);  // [后序遍历钩子位置]}

上面代码中的三个位置，前序遍历、中序遍历、后序遍历，相当于三个“钩子”，允许你在递归遍历的过程中执行一些额外的操作。

下面用一个最简单的例子来说明：

Leetcode 第 226 题，翻转一棵二叉树，示例：

>_<
// 输入     4   /   \  2     7 / \   / \1   3 6   9
// 输出     4   /   \  7     2 / \   / \9   6 3   1

这道题用前序遍历，按照上面的框架代入，还是比较简单的：

>_<js
function invertTree(root) {  if (root === null) return null;  // 前序遍历  [root.left, root.right] = [root.right, root.left];  invertTree(root.left);  invertTree(root.right);  return root;}

我们在前序的位置，将根节点的左右子树进行互换，然后再分别递归翻转左右两棵子树，最终将整棵二叉树进行翻转。

这道题也可以用后序遍历来做，先将根节点的左右子树分别翻转完成，再将二者进行互换，如下：

>_<js
function invertTree(root) {  if (root === null) return null;  invertTree(root.left);  invertTree(root.right);  // 后序遍历  [root.left, root.right] = [root.right, root.left];  return root;}
