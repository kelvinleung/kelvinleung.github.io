说起 JavaScript，一个绕不开的话题肯定是 Prototype，原型。如何去理解原型，得先从“对象”开始讲起。

我们知道，在 Javascript 中，“万物皆对象”，当然这个说法并不严谨。但毫不夸张地说，JS 的确是一门相当“面向对象”的语言。

传统面向对象语言中，对象都是基于“类”创建出来的，这个过程叫“实例化”。类就相当于是对象的设计图（或者说“原型”），它规定了其对应的对象长什么样，有什么属性和方法，是一个“抽象”的东西。而在程序中真正发挥作用的是对象，是一个实实在在的“实例”。因此对象需要通过类来创建，将抽象的东西具体化，这或多或少有些“绕”。

而在 JS 中，虽然有`class`关键词，但实际上这只是“语法糖”，本质上还是一个构造函数（函数也是对象！）。在 JS 中，你甚至可以不通过“类”，就能直接创建一个对象：

```js
const obj = { name: "对象" };

const anotherObj = Object.create(obj);
```
